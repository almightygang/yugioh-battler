# 0.0.4 — Game Loop & Frontend

> **Date:** 2026-07-21  
> **Status:** Pre-alpha / Playable

---

## Overview

v0.0.4 introduces the full gameplay loop: a gacha collection system, team builder, and battle arena wrapped in a single-page web frontend. Players earn currency from battles, spend it on packs to collect monsters, build teams from their collection, and battle NPC opponents.

---

## Features

### Game Loop Architecture

```
Main Menu/Hub
  ├── Shop (Master Pack gacha)
  ├── Collection (monster vault + stat viewer)
  ├── Team Builder (6-slot roster with item assignment)
  └── Arena (5 difficulty stages, NPC battles, rewards)
```

### Player Profile & Inventory (`src/game/player.ts`)
- Persistent save file (`src/game/save.json`) auto-created on first launch
- Currency system (start with 1000cr, earn from arena, spend on packs)
- Monster vault tracks all collected monsters (no dupes)
- All 23 items unlocked by default from a fresh profile
- 10 team slots (save/load/clone/clear)
- Team validation: 3 active slots required, unique-item-clause enforced

### Shop & Gacha (`src/game/shop.ts`)
- Single **Master Pack** (500cr) containing all 8 monster species
- Weighted rarity: legendary monsters are rarer than commons
- **No duplicate pulls** — only unowned monsters are offered
- Refunds if all monsters already collected
- All pulls at **Lv.100**
- No items in packs (all items are free/unlocked)

### Battle Arena (`src/game/arena.ts`)
- 5 stages: Rookie → Intermediate → Advanced → Elite → Champion
- NPC teams generated dynamically at Lv.100 with random items
- Full `BattleEngine` execution (10-turn cap, then HP comparison)
- Rewards scale with difficulty (100–1000cr)
- Combat log displayed in the frontend

### Collection Page (`#collection`)
- Grid of owned monsters sorted alphabetically
- Each card shows sprite, name, Lv.100, element badges
- Click opens **detail modal** with:
  - Monster sprite (with fallback on missing image)
  - Archetype, legendary status, element/type badges
  - Full base stats with color-coded values (high/mid/low)
  - Total BST
  - Ability ID
  - Full learnset

### Team Builder (`#teambuilder`)
- Left panel: 6 slot cards (L/C/R active + B1/B2/B3 bench)
- Right panel: Monster vault and Item vault tabs
- Click a slot → click a monster to assign
- Unique-item clause enforced (same item can't be equipped twice)
- Validation badge: red "Incomplete" or green "Battle Ready"
- Save/Clone/Clear team actions

### Monster Sprites
- Sprites stored as `public/images/monsters/{id}.png`
- Server serves static files from `public/` at `/images/`
- Fallback placeholder displayed when no sprite file exists

## Known Limitations (for v0.0.5)

- No speed-ordering turn queue (moves execute in caller order)
- No double battles / tag-team mechanics
- No network multiplayer
- AI is simple random-pick (no strategy or prediction)
- No battle animations or UI beyond text log
- Effects are code-defined in `effects.ts` — `effects.json` stores only metadata
- No monster evolution/level-up system (all monsters fixed at Lv.100)
- No equipment variety system (all items always unlocked)

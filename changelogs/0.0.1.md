# 0.0.1 — Initial Build

> **Date:** 2026-07-21  
> **Status:** Pre-alpha / Foundation

---

## Overview

Initial build of Yu-Gi-Oh! Battler — a turn-based 3v3 monster battle simulator combining Pokémon-style combat mechanics with Yu-Gi-Oh! theming.

---

## Features

### Core Engine
- Turn-based battle engine with 3v3 slot format
- Pokémon-style damage formula with STAB, type effectiveness, critical hits, and random variance
- 6 target scope patterns (single adjacent, single any, all opponents, allies, everyone-but-self, self)
- Turn advancement with buff/status/field duration management
- Target redirection via Eyes of Blue ability

### Type System
- 7 Element attributes (LIGHT, DARK, FIRE, WATER, EARTH, WIND, NORMAL) with full effectiveness chart
- 9 Monster Types (DRAGON, WARRIOR, SPELLCASTER, SUPPORT, ZOMBIE, FIEND, FAIRY, MACHINE, ROCK)
- 3 move categories (Physical, Special, Status)
- Archetype string tag on monsters for search/targeting

### Database (JSON-driven)
- 8 monsters across 5 archetypes (Blue-Eyes, Dark Magician, Zombie, Megalith)
- 45 moves covering all 7 elements
- 16 abilities including core mechanics and type immunities
- 22 items mapped from Yu-Gi-Oh! cards to Pokémon equivalents

### Graveyard & Banish Zones
- Faint → GY system with revive support
- Permanent exile via banish zone (no normal revival)
- GY-count-scaling abilities

### Status System
- 9 primary statuses (mutually exclusive): Stacked, Pierced, Cursed, Blinded, Vitalized, Burn, Poison, Paralysis
- 5 secondary statuses (stackable, clear on switch): Skill Drained, Charmed, Chained, Life Drained, Tethered
- End-of-turn tick processing for all active statuses
- Status source tracking for recoil and healing attribution

### Life Conversion
- HP-sacrifice move costs (hpCostPercent)
- HP-threshold gating (hpThreshold)
- HP-heal move rewards (healPercent)

### Weather & Fields
- Weather system: Blinding Radiance (LIGHT boost, accuracy penalty)
- Field system: Zombie World (DARK override), Megalith Portal (EARTH boost)
- Fields last 5 turns, override-only (not weather)

### Effect Hooks
- 30 move effect hooks (20 implemented, 10 stubs)
- 16 ability effect hooks (2 implemented, 14 stubs)
- 22 item effect hooks (2 implemented, 20 stubs)

## Technical Stack

- **Language:** TypeScript (ES2020)
- **Module System:** CommonJS
- **Runtime:** Node.js 26.x
- **Build:** tsc via npx
- **Dependencies:** typescript ^7.0.2, ts-node ^10.9.2, @types/node ^26.1.1

## Known Limitations

- Many ability/item effect hooks are stubs awaiting implementation
- No AI opponent logic (engine requires manual move commands)
- No bench/switch mechanic implemented yet
- Single weather type currently defined
- No multi-hit move logic for Dual Wingbeat
- No entry hazards (Stealth Rock) or terrain mechanics
- Burn/Poison/Paralysis status tick damage not implemented
- No stat stage system for ATK/DEF/SPATK/SPDEF/SPE boosts

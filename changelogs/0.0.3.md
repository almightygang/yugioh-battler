# 0.0.3 ‚Äî Tools & Persistence

> **Date:** 2026-07-21  
> **Status:** Pre-alpha / Tooled

---

## Overview

v0.0.3 focuses on developer tooling and data persistence: a full battle-state save/load system, an interactive database editor (both CLI and GUI), and an effects database to complement the code-defined effect hooks. Numerous bug fixes from the 0.0.2 audit are also included.

---

## Features

### SaveManager (`src/engine/SaveManager.ts`)
- `saveToFile(filePath)` ‚Äî Serializes full `BattleState` (ally team, opp team, weather, field, graveyard, banish, hazards, buffs, itemConsumed, turnCount) to JSON
- `loadFromFile(filePath)` ‚Äî Deserializes JSON back into a `BattleState` and restores a `BattleEngine` instance with full state
- `exportState()` / `importState()` ‚Äî In-memory JSON string export/import
- `restoreEngine()` ‚Äî Reconstructs a new `BattleEngine` from a loaded state, re-running on-entry abilities and abilities
- `buffs`, `itemConsumed`, `turnCount` added to `BattleState` schema for complete roundtrip fidelity

### Database Editor CLI (`src/tools/db-editor.ts`)
- Interactive terminal-based CRUD for all 5 databases (abilities, items, moves, monsters, effects)
- Full field-type support: string, number, boolean, enum dropdowns, string[] with tag-style entry, nested objects (baseStats, multiHit)
- Create, read, update, delete entries from the command line
- Search/filter entries by name or ID

### Database Editor GUI (`src/tools/db-server.ts` + `src/tools/db-gui.html`)
- HTTP API server using Node.js `http` module (zero extra dependencies)
- REST endpoints: `GET /api/databases`, `GET /api/databases/:name`, `GET /api/databases/:name/:id`, `POST /api/databases/:name/:id`, `PUT /api/databases/:name/:id`, `DELETE /api/databases/:name/:id`
- `GET /api/schema` ‚Äî Auto-generates field schemas by scanning all entries in each database
- Single-page HTML/CSS/JS frontend served on `http://localhost:3000`
- Database tabs, searchable entry list, full CRUD with dynamic form rendering
- Ctrl+S keyboard shortcut for save

### Auto-Generated Schema
- No hardcoded field definitions ‚Äî the server examines actual data to infer types
- Recognizes: `string`, `number`, `boolean`, `enum` (2‚Äì40 unique values), `object` (recursive nested), and `string[]` (with value suggestions)
- Detects optional vs required fields by entry coverage
- Adding new fields to any JSON database automatically updates the editor

### Effects Database (`src/database/effects.json`)
- 88 entries covering every `effectId` referenced across abilities, items, and moves
- Each entry: `id`, `name`, `description`, `effectType` (move/ability/item)
- Provides a single source of truth for effect metadata alongside the code-defined hooks in `effects.ts`
- Editable through both CLI and GUI editors

## Known Limitations (for v0.0.4)

- No double battles / tag-team mechanics
- No rental/inverse battle formats
- No network multiplayer
- AI is simple random-pick (no strategy or prediction)
- No battle animations or UI beyond console logging
- Effects are code-defined in `effects.ts` ‚Äî the `effects.json` database only stores metadata; adding a new functional effect still requires TypeScript code
